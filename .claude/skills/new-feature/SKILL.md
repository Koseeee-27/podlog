---
name: new-feature
description: 新機能を設計レビューしてから実装する。引数に実装したい機能の説明を渡す
disable-model-invocation: true
argument-hint: "[実装したい機能の説明]"
---

新機能を実装する前に、以下の設計レビューを行ってください。

機能の説明: $ARGUMENTS

手順:
1. **要件整理**: 機能の目的・ユーザーストーリーを確認する
2. **設計提案**:
   - フロントエンド: 必要なコンポーネント・ページ・カスタムフックを列挙
   - バックエンド: 必要なAPIエンドポイント（メソッド + パス）を列挙
   - DB: 必要なテーブル・カラムの変更を列挙
3. **実装順序**: どの順番で実装するか提案する
4. **ユーザーに確認**: 設計で問題ないか確認を取ってから実装に入る

Goのコードを含む場合は、何をしているかコメントと説明を加えること。
