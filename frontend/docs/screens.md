# 画面仕様書

## 概要

PodLog フロントエンドの全画面仕様を定義する。各画面について目的、URL、表示要素、ユーザー操作、認証要否を記載する。

本ドキュメントは以下に基づいて作成されている:
- `docs/requirements.md`（機能要件書）
- `docs/ux-guideline.md`（UX ガイドライン）
- `docs/design-guideline.md`（デザインガイドライン）

### 設計方針

- **フルオープン型**: 閲覧系の画面はすべて認証不要。書き込み操作のみログインを必須とする
- **Google ソーシャルログイン**: メール+パスワード認証は MVP では提供しない
- **SP / PC で UI を分ける**: SP はボトムナビ、PC はヘッダーナビでそれぞれ最適な UX を提供する
- **アプリライクな体験**: SP ではモバイルアプリに近い操作性を目指す

## ナビゲーション

### SP（モバイル）: ボトムナビゲーション

ハンバーガーメニューを廃止し、ボトムナビを採用する。親指で操作しやすく、主要機能にワンタップでアクセスできる。

```
┌──────────────────────────────────────────────────┐
│                                                  │
│  （コンテンツ領域）                                │
│                                                  │
├──────────────────────────────────────────────────┤
│  🏠 ホーム   🔍 探す   ＋ 記録する   👤 マイページ │
└──────────────────────────────────────────────────┘
```

| タブ | アイコン | 遷移先 | 役割 |
|---|---|---|---|
| ホーム | Home | `/` | タイムライン。自分に関連する情報 |
| 探す | Search / Compass | `/discover` | 検索・おすすめ。新しい番組との出会い |
| 記録する | Plus | 記録フロー | 聴いた番組の記録に特化した導線 |
| マイページ | User | `/users/{username}` | 聴取履歴・レビュー・プロフィール |

#### ボトムナビのルール
- タブは **4つ固定**
- アイコン + ラベル（テキスト）を併記
- アクティブタブはローズピンク（`rose-500`）で強調
- 未ログイン時: 「マイページ」タブは「ログイン」に変わり `/login` へ遷移
- 下スクロール時に隠す / 上スクロール時に表示する（コンテンツ領域の確保）

### PC（デスクトップ）: ヘッダーナビゲーション

```
┌──────────────────────────────────────────────────┐
│  PodLog     [🔍 番組を検索...          ]  [Avatar ▾]  │
└──────────────────────────────────────────────────┘
```

| 要素 | 位置 | 内容 |
|---|---|---|
| ロゴ | 左 | 「PodLog」。クリックで `/` へ遷移 |
| 検索バー | 中央 | キーワード入力で `/discover?q=...` へ遷移 |
| アバター | 右（ログイン済み） | クリックでドロップダウンメニュー |
| ログインボタン | 右（未ログイン） | `/login` へ遷移 |

#### アバタードロップダウン（ログイン済み時）

```
┌────────────────────┐
│  表示名             │
│  @username          │
│  ─────────────────  │
│  マイページ          │ → /users/{username}
│  設定               │ → /settings
│  ─────────────────  │
│  ログアウト          │
└────────────────────┘
```

## 2. ログイン

- **URL**: `/login`
- **認証**: 不要
- **目的**: Google アカウントでのログイン / 新規登録

MVP では Google ソーシャルログインのみ提供する。新規登録とログインを同一画面で扱う。

### 表示要素

```
┌──────────────────────────────────────────────────┐
│  PodLog                                          │
├──────────────────────────────────────────────────┤
│                                                  │
│              アカウントにログイン                    │
│                                                  │
│       [G  Google アカウントでログイン]              │
│                                                  │
│       ログインすることで利用規約と                  │
│       プライバシーポリシーに同意します               │
│                                                  │
└──────────────────────────────────────────────────┘
```

- ヘッダーにロゴ「PodLog」を表示。クリックで `/` へ遷移（逃げ道の確保）
- ナビゲーション（ボトムナビ / ヘッダーナビ）は表示しない

### ユーザー操作

| 操作 | 挙動 |
|------|------|
| ロゴ押下 | `/` へ遷移 |
| 「Google アカウントでログイン」押下 | Google OAuth 認証フローを開始。認証成功後、プロフィール未設定なら `/profile/setup` へ、設定済みなら `/` へ遷移 |

### 状態別表示

| 状態 | 表示 |
|------|------|
| 認証処理中 | ボタンが無効化され、ローディング表示 |
| エラー | ボタン下部にエラーメッセージ表示 |
| ログイン済みでアクセス | `/` へリダイレクト |

## 3. プロフィール初期設定

- **URL**: `/profile/setup`
- **認証**: 必要
- **目的**: 新規登録後のプロフィール初期設定（ユーザー名・表示名の必須設定）

### 表示要素

```
┌──────────────────────────────────────────────────┐
│  [ヘッダー / ボトムナビ]                           │
├──────────────────────────────────────────────────┤
│                                                  │

│   はじめに、プロフィールを設定しましょう             │
│                                                  │

### ユーザー操作

| 操作 | 挙動 |
|------|------|

### バリデーション

- ユーザー名: 必須、3〜30文字、英数字とアンダースコアのみ、一意であること
- 表示名: 必須

### 状態別表示

| 状態 | 表示 |
|------|------|
| 未認証 | `/login` へリダイレクト |
| プロフィール設定済み | `/` へリダイレクト |
| ユーザー名が既に使用されている | フィールド下にエラーメッセージ |
| エラー | フォーム上部にエラーメッセージ表示 |
| 送信中 | ボタンが無効化される |

### 人気の番組セクション（MVP）

- レビュー件数の多い番組を一覧表示する（MVP では簡易的な形でスタート）
- 検索キーワードが空の時に表示。検索中は検索結果に切り替わる

### 検索結果カードの表示項目

- 番組アートワーク
- 番組タイトル
- 配信者名
- 平均評価、レビュー件数

### ユーザー操作

| 操作 | 挙動 |
|------|------|
| 検索入力 | デバウンス後にアプリ内 DB を検索。結果を表示 |
| 番組カード押下 | `/podcasts/[id]` へ遷移 |
| 「番組の追加をリクエストする」押下 | 未ログインなら `/login` へ遷移。ログイン済みなら追加リクエストダイアログを表示 |

```

│                                          │
└──────────────────────────────────────────┘
```

### 状態別表示

| 状態 | 表示 |
|------|------|
| 初期（未検索） | 人気の番組セクションを表示 |
| ローディング | 検索バーにスピナー表示 |

| エラー | エラーメッセージ表示 |

### 段階的な拡充

| フェーズ | 探す画面の内容 |
|---|---|
| MVP | キーワード検索 + 人気番組の一覧 |
| Phase 2 | ランキング（週間・月間）、ジャンル別ブラウズ |
| Phase 3 | パーソナライズされたおすすめ（レコメンドエンジン） |

┌──────────────────────────────────────────┐
│ あなたのレビュー                          │
│                                          │
│ 評価: ★★★★☆                              │
│ コメント: コメントテキスト...              │
│                                          │

└──────────────────────────────────────────┘
```

### ユーザー操作

| 操作 | 挙動 |
|------|------|
| 番組名押下 | `/podcasts/[id]` へ遷移 |
| 「聴いたにする」押下 | 聴取記録を追加、ボタンが「聴いた」に変化 |
| 「聴いた」押下 | 聴取記録を取り消し、ボタンが「聴いたにする」に変化 |
| 星をクリック | 評価を選択（1〜5） |
| 「投稿する」押下 | レビューを作成、レビュー一覧を更新 |
| 「編集する」押下 | 自分のレビューを編集モードに切り替え |
| 「削除する」押下 | 確認表示後、レビューを削除 |
| レビュー内のユーザー名 / アバター押下 | `/users/[username]` へ遷移 |
| 「もっと見る」押下 | 次のページのレビューを読み込み |
| 「ログインして記録する」押下（未ログイン時） | `/login` へ遷移 |
| 「ログインしてレビューを書く」押下（未ログイン時） | `/login` へ遷移 |

### バリデーション

- 評価: 必須（1〜5の整数）
- コメント: 任意、最大1000文字

### 状態別表示

| 状態 | 表示 |
|------|------|

| エピソード未発見 | 「エピソードが見つかりませんでした」 |

## 記録フロー

ボトムナビの「＋ 記録する」タブから始まる、聴いた番組の記録に特化した導線。

```
「＋ 記録する」タップ
  │
  ├─→ 番組検索（モーダル or 専用画面）
  │     │
  │     └─→ 番組を選択
  │           │
  │           └─→ エピソード選択
  │                 │
  │                 └─→ /episodes/[id] へ遷移
  │                       │
  │                       ├─→ 「聴いたにする」で記録
  │                       └─→ レビューも書ける
```

### 記録フローの UX 方針

- 「探す」画面とは目的が異なる（ブラウズではなく、記録に特化）
- 最短 3 タップで記録完了を目指す: 「記録する」→ 番組選択 → エピソード選択
- MVP では `/discover` の検索にフォールバックし、段階的に専用 UI を構築する

## 認証フロー

```
未ログイン
  │
  ├─→ /（トップ）→ 「Google で始める」→ Google 認証
  │                                        │
  │                                        ├─→ 初回 → /profile/setup → /
  │                                        └─→ 2回目以降 → /
  │
  ├─→ /login → Google 認証 → 同上
  │
  ├─→ /discover, /podcasts/[id], /episodes/[id], /users/[username]
  │     → 認証なしで閲覧可能
  │     → 書き込み操作時に /login へ誘導
  │
  └─→ /settings, /settings/profile にアクセス → /login へリダイレクト

ログイン済み・プロフィール未設定
  │
  └─→ 書き込み操作時 → /profile/setup へリダイレクト

ログイン済み・プロフィール設定済み
  │
  └─→ 全ページにアクセス可能、全操作が利用可能
```
